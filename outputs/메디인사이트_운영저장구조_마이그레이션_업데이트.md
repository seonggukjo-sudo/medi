# 메디인사이트 운영 저장 구조

## 저장 원칙

- 업로드 API에서 R2에 원본 CSV 저장
- D1에 업로드 배치와 파일 메타데이터 저장
- 검증을 통과한 CRM, 예약, 내원, 결제, 광고 데이터를 병원별 업무 테이블에 반영
- 직접 수정값은 원천 데이터를 변경하지 않고 보정값과 감사 이력으로 별도 관리

## 주요 저장소

- `R2 UPLOADS`: 업로드한 원본 CSV와 재처리 가능한 원본 파일
- `D1 DB`: 업로드 배치, 파일 메타데이터, 업무 데이터, 사용자, 설정, KPI 스냅샷, 감사 이력
- `hospital_settings`: 병원별 KPI 목표, AI 분석 기준, 화면 및 연동 설정

## 데이터 관리 검증

- 선택 기간의 기간 합계와 세부 합계를 자동 대사
- 중복 식별키, 필수값 누락, 연결되지 않은 예약·내원·결제 데이터를 경고
- 원천 합계, 수동 보정, 최종 반영 합계를 분리 표시
- 업로드 파일명, 업로드자, 업로드 시간, 적용 기간 및 처리 상태를 보존

## 복구와 감사

- 원본 CSV는 R2에서 다시 처리할 수 있습니다.
- 직접 수정은 수정자, 수정 시각, 사유, 이전값과 변경값을 D1 감사 이력에 기록합니다.
- 삭제 또는 보정 이전 상태는 감사 이력을 이용해 복원합니다.
